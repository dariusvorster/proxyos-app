# Environment Variables

All variables are set in the Docker container environment. Variables marked **Required** must be set or the container will not start correctly.

---

## Core

| Variable | Default | Required | Description |
|---|---|---|---|
| `PROXYOS_SECRET` | — | **Yes** | Secret used for session signing and internal crypto. Must be at least 32 characters. Generate with `openssl rand -hex 32`. |
| `PROXYOS_DB_PATH` | `/data/proxyos/proxyos.db` | No | Path to the SQLite database file. Must be on a persistent volume. |
| `PROXYOS_ACCESS_LOG` | `/data/proxyos/access.log` | No | Path Caddy writes JSON access log entries to, and the collector tails. Must be on a persistent volume. |
| `PROXYOS_MODE` | `standalone` | No | Operational mode. One of: `standalone`, `central`, `node`, `central+node`. |

---

## Caddy

| Variable | Default | Required | Description |
|---|---|---|---|
| `CADDY_ADMIN_URL` | `http://localhost:2019` | No | URL of the Caddy Admin API. Do not change unless you are running Caddy externally. |
| `CADDY_BASE_CONFIG_PATH` | `/etc/caddy/base-config.json` | No | Path to the base Caddy JSON config loaded at startup before any routes are applied. |

---

## TLS / Certificates

| Variable | Default | Required | Description |
|---|---|---|---|
| `CLOUDFLARE_API_TOKEN` | — | No | Cloudflare API token with `Zone:DNS:Edit` permission. Required to use DNS-01 TLS mode on any route. |

---

## Federation

| Variable | Default | Required | Description |
|---|---|---|---|
| `PROXYOS_FEDERATION_PORT` | `7890` | No | Port the federation WebSocket server listens on. Used by central and `central+node` instances. Must be reachable by nodes. |
| `PROXYOS_CENTRAL_URL` | — | Node mode | WebSocket URL of the central instance. Example: `wss://central.example.com:7890`. Required when `PROXYOS_MODE=node` or `central+node`. |
| `PROXYOS_AGENT_TOKEN` | — | Node mode | Enrollment token generated by the central instance. Required when `PROXYOS_MODE=node`. |
| `PROXYOS_AGENT_NAME` | — | No | Human-readable name for this node, shown in the central UI. Optional but recommended. |

---

## Volume paths (not env vars)

These are filesystem paths that must be mounted as Docker volumes. They are not environment variables but are documented here for reference.

| Path | Purpose |
|---|---|
| `/data/proxyos` | SQLite database and access log. Mount a persistent volume here. |
| `/data/caddy` | Caddy TLS state: issued certificates, ACME account keys. Mount a persistent volume here. |
| `/config/caddy` | Caddy config persistence. Mount a persistent volume here. |
| `/var/run/docker.sock` | Docker socket. Required for the Docker Scanner feature. Bind-mount from host. |

---

## Example: full compose env block

```yaml
environment:
  # Required
  PROXYOS_SECRET: "change-me-to-a-32-plus-char-random-string"

  # Paths (only override if your volume layout differs)
  PROXYOS_DB_PATH: /data/proxyos/proxyos.db
  PROXYOS_ACCESS_LOG: /data/proxyos/access.log

  # Caddy (leave at defaults unless running Caddy externally)
  CADDY_ADMIN_URL: http://localhost:2019
  CADDY_BASE_CONFIG_PATH: /etc/caddy/base-config.json

  # Mode
  PROXYOS_MODE: standalone

  # Optional: Cloudflare DNS-01
  # CLOUDFLARE_API_TOKEN: your-cloudflare-api-token

  # Optional: federation (central)
  # PROXYOS_FEDERATION_PORT: "7890"

  # Optional: federation (node)
  # PROXYOS_CENTRAL_URL: wss://central.example.com:7890
  # PROXYOS_AGENT_TOKEN: enrollment-token-from-central
  # PROXYOS_AGENT_NAME: node-name
```
